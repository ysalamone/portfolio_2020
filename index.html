JANVIER 2019 – AVRIL 2020
Projet de BTSNew World
Capture d'écran
Description du projet

Création d’une plateforme de drive équitable sous 3 aspects :

  • Un site internet permettant l’inscription en tant que producteur, point relais ou consommateur, et permettant de gérer son compte (ses informations personnelles, ses produits en ligne et les points retraits sélectionnés si l’on est producteur) ;
  • Un logiciel d’administration back-office permettant la gestion des produits et des utilisateurs du site ;
  • Un logiciel permettant de générer automatiquement un catalogue hebdomadaire personnalisé, montrant les produits disponibles dans les points relais sélectionnés par le consommateur.

Langages & techniques utilisés
  • HTML
  • CSS
  • Bootstrap
  • JavaScript & JSON
  • AJAX
  • PHP
  • SQL & mariaDB
  • Qt
JANVIER 2020 – FÉVRIER 2020
Stage de 2ème année de BTSPastèque
Capture d'écran
Description du projet

Étude du fonctionnement des logiciels de caisse "Pastèque" (écosystème composé d'un serveur avec API, d'une interface d'administration en PHP et d'un client desktop Java).

L'objectif était de personnaliser cet écosystème, afin de synchroniser Pastèque avec la base de données des spectacles proposés par une ville (pour y implémenter les catégories, les spectacles, les séances et les tarifs proposés).

  • Dans le serveur, création d'une route API de récupération de données depuis le back-office du client :
    • Création des nouvelles entrées ;
    • Mise à jour des entrées existantes ;
    • Suppression des entrées obsolètes ;
  • Dans l'interface d'administration, création d'un panneau dédié à la synchronisation de ces données.

Langages & techniques utilisés
  • PHP (orienté objet)
  • ORM Doctrine
  • Moteur de rendus Twig
  • Micro-framework Silex
  • JSON
  • SQL
MAI 2019 – JUILLET 2019
Stage de 1ère année de BTSPharmacie amie
Capture d'écran
Description du projet

Création d'un portage du site pharmacieamie.fr sur iOS et Android à l'aide de React Native.

Implémentation des fonctionnalités suivantes, avec dialogue entre la WebView affichée et l'application native :

  • Récupération de la géolocalisation utilisateur (via le GPS du téléphone), chargement du site avec affichage des pharmacies à proximité ;
  • Détection du premier lancement de l'application ;
  • Affichage de notifications locales ;
  • Affichage de notifications distantes générées par Firebase ;
  • Détection de la sortie du site et du téléchargement de fichiers (sortie de l'application).

Langages & techniques utilisés
  • React Native
  • JavaScript & JSON
